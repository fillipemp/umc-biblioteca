1 INTRODUÇÃO

Este manual mostra como usar o Sistema de Gerenciamento de Biblioteca (SGBP), desenvolvido em Java com Spring Boot, Thymeleaf e MongoDB. O sistema controla o acervo de uma biblioteca: cadastro de livros e leitores, reservas, empréstimos e devoluções.

O sistema roda no navegador, não precisa instalar nada além do Java. Todas as telas são acessadas pelo endereço http://localhost:8080 após iniciar a aplicação.

Código-fonte: [INSERIR LINK DO GITHUB]

2 COMO ACESSAR O SISTEMA

Antes de acessar, a aplicação precisa estar rodando. No Windows, execute o arquivo iniciar.ps1 na raiz do projeto clicando com o botão direito → "Executar com PowerShell". Aguarde a mensagem de inicialização e então abra o navegador em:

http://localhost:8080

O sistema abre direto na tela de login. Use as credenciais abaixo para o primeiro acesso:

3 TELA INICIAL (DASHBOARD)

Após fazer login, o sistema mostra o dashboard. Ele exibe quatro totais em destaque: quantos livros estão cadastrados, quantos leitores, quantos empréstimos estão abertos e quantas reservas ainda estão pendentes. É uma visão geral rápida da situação da biblioteca.

A navegação entre as telas é feita pelo menu lateral esquerdo, que fica visível em todas as páginas.

4 LIVROS

4.1 Ver e buscar livros

Clique em Livros no menu lateral. A tela mostra todos os livros cadastrados em uma tabela com: título, autor, ISBN, ano, quantidade total de exemplares e quantidade disponível para empréstimo.

Para buscar um livro específico, digite qualquer palavra no campo de busca e a lista filtra automaticamente.

6 RESERVAS

A reserva serve para o leitor garantir um livro antes de ir buscá-lo. Ela não desconta o estoque — só o empréstimo faz isso. Cada reserva tem validade de 3 dias; depois desse prazo ela expira automaticamente.

6.1 Fazer uma reserva

  1. No menu lateral, clique em Reservas;
  2. Clique em + Nova Reserva;
  3. Escolha o livro na lista — só aparecem livros com exemplares disponíveis;
  4. Escolha o leitor;
  5. Clique em Criar Reserva.

7 EMPRÉSTIMOS

O empréstimo registra quando um leitor leva um livro. Ao confirmar, o sistema desconta 1 exemplar disponível e define automaticamente o prazo de devolução para 14 dias.

Para registrar um empréstimo:

  1. Clique em Empréstimos no menu lateral;
  2. Clique em + Novo Empréstimo;
  3. Selecione o livro (só aparecem livros com exemplares disponíveis);
  4. Selecione o leitor;
  5. Clique em Registrar Empréstimo.

8 DEVOLUÇÕES

A tela de devoluções lista só os empréstimos que ainda estão em aberto (ativos ou em atraso). Para registrar a devolução de um livro:

  1. Clique em Devoluções no menu lateral;
  2. Encontre o empréstimo na lista;
  3. Clique em Registrar Devolução e confirme.

Ao confirmar, o sistema soma 1 de volta ao estoque disponível do livro e marca o empréstimo como Devolvido.

9 OPERADORES DO SISTEMA

Operadores são as pessoas que têm login no sistema — os funcionários que acessam e gerenciam a biblioteca. Esse módulo fica na seção Administração do menu lateral, em Operadores.

Cada operador tem nome, e-mail, senha (criptografada) e um perfil: Administrador (acesso total) ou Operador (acesso padrão).

Figura 11 — Lista de Operadores [ Cole aqui o print da lista de operadores — http://localhost:8080/users ]

Figura 11 — Lista de operadores do sistema. Fonte: o autor (2026).

9.1 Cadastrar operador

Clique em + Novo Operador, preencha nome, e-mail, perfil e senha (mínimo 6 caracteres). Confirme a senha no segundo campo e salve.

Figura 12 — Formulário de Novo Operador [ Cole aqui o print do formulário — http://localhost:8080/users/new ]

Figura 12 — Formulário de cadastro de operador. Fonte: o autor (2026).

9.2 Editar operador

Clique em Editar na linha do operador. Se não quiser trocar a senha, deixe os campos de senha em branco — o sistema mantém a senha atual. Para trocar, preencha a nova senha e confirme.
